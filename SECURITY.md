# Security Policy

## Reporting a Vulnerability

Please do not open public issues for security vulnerabilities.

Use GitHub Security Advisories to report vulnerabilities privately:

- Go to the repository Security tab.
- Use "Report a vulnerability".
- Provide a clear description, impact, and reproduction details.

If private reporting is temporarily unavailable, open a GitHub Discussion asking for a private contact channel without disclosing sensitive details.

## Disclosure Process

- We will acknowledge valid reports as soon as possible.
- We target an initial triage response within 5 business days.
- Fix timelines depend on severity, complexity, and release coordination.
- We will coordinate disclosure timing with reporters when possible.

## Scope

This policy covers source code and release artifacts in this repository.
